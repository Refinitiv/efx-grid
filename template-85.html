Initializing data in Atlas Blotter

Data will be defined through staticDataRows. Given data format should be array of object maps. 2D Array data format is also accepted by Atlas Blotter. Atlas Blotter will automatically detect the format type from the given data.

var configObj = {
    //...
    staticDataRows: [
        {"Field 1": 1, "Field 2": 2}, // First row
        {"Field 1": 3, "Field 2": 4}, // Second row
        //...
    ],
    //...
};

var configObj2 = {
    //...
    columns: [
        { field: "Field 1" },
        { field: "Field 2" },
        // ...
    ],
    //...
    staticDataRows: [
        [1, 2], // Fields will be taken from column configuration
        [3, 4],
        //...
    ],
    //...
};

Static data can also be defined in values property in the row configuration object. This is useful when we want to merge real-time data with the static data. 

var configObj = {
    //...
    rows: [
        { ric: "ABC", values: {"Field 1": 1, "Field 2": 2}},
        { ric: "ABC", values: {"Field 1": 3, "Field 2": 4}},
    ],
    //...
};

For more details about initilizing data see Reseting/Clearing data page.
