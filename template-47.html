Overview

Grid extensions provide enhancements for the Grid element. This allows for greater extensibility without increasing the size of the main code base.

Installation

npm install PROJECT_NAME

PROJECT_NAME is an extension's project name (tr-grid-xxx). In this document the project name of each extension is in parentheses next to the extension-name on the top of each extension page.

For example

npm install tr-grid-auto-tooltip

Setup guide

Traditional usage

To use the extension as an external static library, install it from npm. Then include the library file on your HTML page. The path may vary depending on your project structure.

<head>
  <script src="node_modules/PROJECT_NAME/dist/PROJECT_NAME.min.js"></script>
</head>

For example

<head>
  <script src="node_modules/tr-grid-auto-tooltip/dist/tr-grid-auto-tooltip.min.js"></script>
</head>

ES6 module usage

To use the extension, install it from npm and deduplicate the installed packages.

npm install PROJECT_NAME
npm dedupe

Then, use import syntax to import the extension into your app. You can use one of the following formats. They are all equivalent.

Using a shorthand to refer to the extension file

import { MODULE_NAME } from "PROJECT_NAME/es6";

// For example
import { AutoTooltip } from "tr-grid-auto-tooltip/es6";

Using a short form (opting out node_modules directory) to refer to the extension file

import MODULE_NAME from "PROJECT_NAME/es6/MODULE_NAME.js";

// For example
import AutoTooltip from "tr-grid-auto-tooltip/es6/AutoTooltip.js";

Using full file path to refer to the extension file. This format is natively supported by all modern browsers. It can be used without bundling or pre-processing. 

import { MODULE_NAME } from "./node_modules/PROJECT_NAME/es6/MODULE_NAME.js";

// For example
import { AutoTooltip } from "./node_modules/tr-grid-auto-tooltip/es6/AutoTooltip.js";

Basic usage

The simplest way to add extensions to Grid is by passing the extension's instances through the extensions property in the configuration object.

var grid = document.getElementById("grid");
grid.config = {
  // main option
  extensions: [
    EXTENTION_INSTANCE
  ]
};

There are two kinds of extension options, the main option and the column-specific option.

The main option is an option declared as a first-level property. Generally, the main option's attribute name is the extension's name but starting with a lower case letter. For example, Column Resizing Extension has columnResizing as its main option name.

The column specific option is defined in the column option under the columns property of Grid's configuration.

Details about both kinds of options are available on each extension's page.

Example of an extension option:

import { ColumnResizing } from "tr-grid-column-resizing/es6";

var grid = document.getElementById("grid");
grid.config = {
    // main option
    columns: [
        // column-specific option
        {
            name: "Number",
            field: "number",
            noResizing: true // column's specific option
        }
    ],

    // extension option
    columnResizing: { 
        preserveProportion: true,
        preserveGridSize: true,
        columnResized: function(e) {} 
    },
    
    extensions: [
        new ColumnResizing() // extension instance needs to be added for the options above to take effect
    ]
};

Adding an event listener to extension's custom event

Some extensions provide and dispatch custom events, allowing custom behaviors to be performed. For example, ColumnResizing extension will dispatch columnResized event after a user has adjusted column size by drag and drop actions. By listening to columnResized event, you can obtain the new column size and perform some custom tasks on the new value. Data or information related to the event will be provided through the event argument object and varied depending on the custom event. There are two ways to add an event listener to the extensions.

Adding event listener through extension's option

An event name inside extension's option will be used as a shortcut for adding an event listener to the event. For example, to add an event listener to columnResized event, you can specify a function on columnReized property, as shown below.

var grid = document.getElementById("grid");
grid.config = {
    columnResizing: { 
        columnResized: function(e) { // Some information is provided on the event argument object
            console.log(e.colIndex, e.width, e.scalability);
        }
    }
    extensions: [
        new ColumnResizing()
    ]
};

Adding event listener directly to extension instance

You can use addEventListener method on extension instance to add an event listener. For Grid plugins (an old variation of Grid extensions), you can use listen method. Using these methods is the same thing as specifying an event listener in the extension's option.

import { ColumnResizing } from "tr-grid-column-resizing/es6";

var colResizingExt = new ColumnResizing();
colResizingExt.addEventListener("columnResized", function(e) {
    console.log(e.colIndex, e.width, e.scalability);
});

var grid = document.getElementById("grid");
grid.config = {
    extensions: [
        colResizingExt // The same extension instance still need be added to grid
    ]
};

Required dependencies

Some extensions require ELF-elements to be installed as part of their dependencies.

For details on how to install and use the ELF-element see the ELF Installation document.

Checkbox Extension

The Checkbox Extension supports both the native checkbox and the coral-checkbox. To use with the coral-checkbox, the following dependency is also required:

npm install @elf/coral-checkbox

Then import to your project index.js.

import "@elf/coral-checkbox";
import "@elf/elf-theme-halo/dark/coral-checkbox"; // Can be any theme.

Range Bar Extension

The Range Bar Extension provides graphical bar for last value ranking visualization based on low and high value with used sapphire-led-gauge. To use the sapphire-led-gauge, the following dependency is also required:

npm install @elf/sapphire-led-gauge

Then import to your project index.js.

import "@elf/sapphire-led-gauge";
import "@elf/elf-theme-halo/dark/sapphire-led-gauge"; // Can be any theme.

Filter Input Extension

The Filter Input Extension supports various types of input elements. These can be archived by specifying the type property through FilterInput option. The ELF component for each type of inputs is shown below:

Type Required ELF-Element
default coral-search-field
number coral-number-field
select coral-select
dropdown coral-select
date emerald-datetime-picker

Note: there is no need to specify a type for the default input element.

Additional dependencies are required according to the type of input. For example, if specified type is date, the additional dependency is emerald-datetime-picker.

Pagination Extension

The Pagination Extension does not provide any UI. You will need to install @elf/emerald-pagination separately for the UI. The extension will automatically wire the functionalities to the UI element once it is presented.

The pagination element is completely independent from the grid element and the Pagination Extension. So, any customization and feature request regarding the element has to be sent to the ELF team. The documentation for the element can be found on ELF's Emerald Pagination page.
