Accessibility in Grid

This page summarizes all the accessibility features Grid has. This page also includes how Grid conforms to Web Content Accessibility Guidelines (WCAG).

Adaptable

Orientation

  • Grid does not restrict its view and operation to a single display orientation, such as portrait or landscape.
  • Grid column width can be dinamically adjusted to fill the available space. See this page for more information.
  • If there is not enough space to display all of its content, horizontal and/or vertical scrollbars can be displayed.

Identify Purpose

  • Grid adds "grid" and "gridcell" role to its corresponding focusable elements.
  • aria-label attribute is added to the elements created by Checkbox Extension, Filter Input Extension, and Filter Dialog to provide their purpose for assistive technology.

Distinguishable

Contrast (Minimum)

  • Heatmap Extension ensures that the color contrast ratio between text and background colors is at least 4.5 or greater.
  • The calculation of color contrast ratio is based on the WCAG relative luminance definition.

Resize text

  • Text content in Grid can be resized without assistive technology up to 200 percent without loss of content or functionality.
  • Zoom Extension allows Grid to be zoomed in and out without affecting other region in the document.

Keyboard Accessible

No Keyboard Trap

  • Grid and its content do not "trap" keyboard focus within subsections of content on a Web page. See this page for more information about keyboard navigation.

Enough Time

No Timing

  • Users face no time limit when using In-Cell Editing Extension, Filter Dialog, Column Selection Dialog, and all other Grid's features.

Seizures

Three Flashes or Below Threshold

  • Grid does not contain anything that flashes more than three times in any one second period, or the flash is below the general flash and red flash thresholds.
  • Blinking or flash interval can still be adjusted.

Navigable

Focus Order

  • In Grid, users can navigate sequentially through focusable content. See this page for more information about keyboard navigation.

Focus Not Obscured (Minimum)

  • When focus leaves the Filter Dialog, the dialog is automatically hidden.
  • The dialog can also be hidden by use of the Escape key.

Input Modalities

Concurrent Input Mechanisms

  • Grid does not restrict use of input modalities available on a platform. Users are free to choose different ways of inputting content.

Target Size (Minimum)

  • The interactive icons created by Row Filtering Extension, Column Stack Extension, Row Grouping Extension, and Row Segmenting Extension are adjusted to have 20 by 20 CSS pixels target size for pointer inputs.

Input Assistance

Redundant Entry

  • The filter information previously entered by or provided to the user is retained by Filter Dialog.
  • When editing Grid's cell, the cell content is auto-populated on the input element by In-Cell Editing Extension.
