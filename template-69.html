Row Dragging

The Row Dragging Extension allows the user to drag and reorder selected rows.

Installation and how to import to a project

Installation examples and details of how to import the extension to a project are available on the Overview page.

Custom dragbox

This allows users to customize the dragBox by providing a dragBoxRenderer option. It is not enabled initially.

var gridConfig = {
    ...
    columns: [
        { name: "Column 1", field: "col1" },
        { name: "Column 2", field: "col2" },
        { name: "Column 3", field: "col3" },
        { name: "Column 4", field: "col4" },
    ],
    rowDragging: {
        dragBoxRenderer: function(e) {
            e.dragBox.textContent = e.dataRow.col1;
        }
    }
    ...
}

Draging a row to the bottom of Grid

You can drag a row to the bottom of Grid by moving the mouse over the bottom half of the last row. The dragged row will be moved and put as the last row. Sometimes, the bottom half of the row can be small and difficult to place your mouse over. You can try to expand Grid's area beyond the last row to mitigate the issue, as the extension allows any drag and drop operation within Grid's area.

Drag and drop content outside of Grid's area

The Row Dragging Extension can be used together with the RowSelectionPlugin to allow users to drag a selected row and drop it to a target outside of a Grid. Then, add dragEnd handler to perform content copying of the selected rows as illustrated below:

html hr {
    margin: 5px;
}
#drop_target {
    height: 200px;
    width: 100%;
}

<big>Try select a row and then drag the selected row to the text box below</big>

<atlas-blotter id="grid"></atlas-blotter>
<hr>
<textarea id="drop_target" placeholder="Drop selected row(s) here"></textarea>

function onDragEnd(e) {
    var dropTargetEl = document.getElementById("drop_target");
    if (e.dropTarget !== dropTargetEl) {
        return;
    }
    var rowSelectionExt = e.srcGrid.getPlugin("RowSelectionPlugin");
    var rowIndices = rowSelectionExt.getSelectedRows();
    
    var text = "No Selected Row";
    if (rowIndices) {
        var rows = grid.api.getMultipleRowData(rowIndices);
        text = JSON.stringify(rows, null, 2)
    }
    dropTargetEl.value = text;
}

        { name: "Industry", field: fields[5] }
    ],
    staticDataRows: records,

        new tr.RowSelectionExtension(), new tr.RowDraggingExtension()

grid.config = configObj;

Undroppable area/rows

The Row Dragging extension has a default behavior for row dragging, which is accompanied by a void icon when a user attempts to drag a row to another location. However, there are certain restrictions in place that forbidden the dragging of certain row types to other locations. For instance, some row types cannot be dragged, while others cannot be dragged to certain areas. To ensure seamless navigation, it is essential to be mindful of these restrictions. Therefore, we have compiled a list of available Row Types, which can be found in the table below.

From/To Row types Content Chain Constituent Group header Group member Subgroup header
Content ✓ ✓ ✗ ✓ ✓ ✗
Chain ✓ ✓ ✗ ✓ ✗ ✗
Constituent ✗ ✗ ✗ ✗ ✗ ✗
Group header ✓ ✓ ✗ ✓ ✗ ✗
Group member ✓ ✓ ✗ ✓ ✓ ✗
Subgroup header ✗ ✗ ✗ ✗ ✗ ✗

Note: Members of the Subgroup header type can only be moved within their group.

Migrating from built-in DragableContentPlugin

Due to the built-in libraries being migrated out from the Core Grid to extensions, built-in extension will be deprecated and will no longer be maintained.

The differences between the legacy (built-in) version of Row Dragging and the new extension can be seen in the examples below

Note: The Row Dragging Extension is backward compatible with the old built-in (with some breaking changes). Therefore, the new implementation is recommended.

Built-in drag box manipulation example

var gridConfig = {
    ...
    rowReorder: {
        dragBoxRenderer: function (dragEvent, dragBox, dataRows) {
            dragBox.setContent("test");
        }
    },
    ...
}

New extension's drag box manipulation example

var gridConfig = {
    ...
    rowDragging: {
        dragBoxRenderer: function (e) {
            // e.dragBox
            // e.dataRows
            e.dragBox.textContent = "test";
        }
    },
    ...
}

For the new implementation, you need to change to the rowDragging option. And the dragBoxRenderer receives an event only as an argument.

beforeDataMoved

Fired after mouseup and before the data/row is about to be moved. You can check the current drag and drop states, cancel operation, or apply custom tasks in this event.
Type:
Object
Properties:
Name Type Attributes Description
rowIndex number Row index of the first row to be dragged
srcRowIndices Array.<number> Row indices of the selected rows
srcRowIds Array.<string> Row ids of the selected rows
srcRowTypes Array.<string> Row types of the selected rows
consolidatedSrcRowType string "MIXED" type is set if there are two or more row types in the srcRowTypes
destRowIndex number Row index of the drop area
destRowId string Row id of the drop area. This will be empty string for the end of grid
destRowType number Row type of the drop area. This will be empty string for the end of grid

Example:
beforeDataMoved: function(e) {
  let srcRowType = e.srcRowTypes[0];
  let destRowType = e.destRowType;
  if(srcRowType === "GROUP_MEMBER" && destRowType === "CONTENT") {
    e.cancel = true; // Prevent a group member row from moving outside of its own group
  }
}
