Atlas Blotter to EFX Grid

After migrating to Atlas Blotter, you can now proceed to migrate to EFX Grid. Since Element Framework v4 has several limitations, we recommend transitioning to Element Framework v6, which introduces more features and functionality. For instance, in Element Framework v6, we have begun supporting Accessibility. Therefore, we strongly recommend migrating to EFX Grid to leverage the capabilities of Element Framework v6.

Overview

On this page, our focus is on migrating from Atlas Blotter to EFX Grid. We recommend migrating Element Framework before migrating to EFX Grid.

Prerequisites

  • Migration to Atlas Blotter: We suggest migrating from Emerald Grid to Atlas Blotter first and then proceeding from Atlas Blotter to EFX Grid. However, you can directly migrate from Emerald Grid to EFX Grid without going through Atlas Blotter. Just be aware that there may be some errors, which you can learn about on the Migration page.
  • Migrate Element Framework from v4 to Element Framework v6 or Element Framework v7
  • Follow the Upgrade Element Framework Guide
  • Understand the Compatibility of bundle tools. For example, in v6, we do not directly support webpack4; if you want to use it with webpack4, you'll need to configure some extra settings in Babel. However, it supports webpack5 as well.

Installation

Ensure you have installed Element Framework v6 to load themes and elements. In Element Framework v6, we have changed the package to an npm package. For EFX Grid, the package is @refinitiv-ui/efx-grid. You can install this package using the following command: npm i @refinitiv-ui/efx-grid

Setup Guide

Use the import syntax to import Atlas Blotter, its dependent elements, and their themes into your app. EFX Grid has a different path from Atlas Blotter, so adjustments are needed.

Grid and Themes

In Atlas Blotter, we have the following imports:

// Import native styles for typography, CSS variables, etc.
import '@elf/elf-theme-halo/dark/imports/native-elements'; 
import '@elf/atlas-blotter';
import '@elf/atlas-blotter/themes/halo/dark/atlas-blotter';

For EFX Grid, they need to be changed to:

// Import native styles for typography, CSS variables, etc.
import "@refinitiv-ui/halo-theme/dark/imports/native-elements";
// Import the element and its Halo dark theme
import "@refinitiv-ui/efx-grid";
import "@refinitiv-ui/efx-grid/themes/halo/dark";

Extensions

In Atlas Blotter, each extension has its own package. The imports look like this:

// import { MODULE_NAME } from "PROJECT_NAME/es6";
import { AutoTooltip } from "tr-grid-auto-tooltip/es6";
import { ColumnResizing } from "tr-grid-column-resizing/es6";
import { ColumnGrouping } from "tr-grid-column-grouping/es6";

For EFX Grid, it's as follows:

// import { EXTENSION_NAME } from '@refinitiv-ui/efx-grid/extensions';
import { AutoTooltip, ColumnResizing, ColumnGrouping } from "@refinitiv-ui/efx-grid/extensions";

As you can see, now we can import extensions from only one path.

Formatters

In Atlas Blotter, each formatter has its own package, which is @grid/formatters. When importing, we have to specify the file name, like this:

import CoralButtonFormatter from "node_modules/@grid/formatters/es6/CoralButtonFormatter.js";
import CoralSelectFormatter from "node_modules/@grid/formatters/es6/CoralSelectFormatter.js";

For EFX Grid, it can be used in only one path, similar to extensions:

import { EFButtonFormatter, EFSelectFormatter } from "@refinitiv-ui/efx-grid/formatters";

Please be careful with these formatters. Element Framework has changed elements from v4 to v6, so the formatters will follow the ELF element. Please follow Predefined Formatter.

Element Usages

For Atlas Blotter, we use <atlas-blotter></atlas-blotter>, while for EFX Grid, we use <efx-grid></efx-grid>.

Post-Migration Reminder

  • Serve Grid: Ensure that the server application runs correctly and that the grid functions as expected.
  • Check CSS Classes: Remember to update tags in CSS classes accordingly.
  • Remove Atlas Blotter Dependencies: Remove any unnecessary dependencies related to Atlas Blotter.
  • Update @refinitiv-ui/efx-grid
  • Add Unit Tests (if necessary): Include unit tests to ensure that all modified methods are functioning correctly.

Summary

If you can serve the application now, congratulations! You have successfully migrated to EFX Grid. Thank you for following.
