Creating or adding data

Adding static data rows to Atlas Blotter

To add static data at the initilization, you can specify your data to staticDataRows property on the configuration object, as shown below:

var configObj = {
    staticDataRows: [
        {"Field 1": "a", "Field 2": "b", "Field 3": "c"},
        {"ABC": "d",     "Field 2": "e", "XYZ": "f"}
    ] // Two rows will be shown
};

To add static data rows at runtime, you can use addStaticDataRows()

Adding real-time rows to Atlas Blotter

In Atlas Blotter, each row will connect to a real-time data source. So, it contains more information than just data content, unlike rows in Emerald Grid. Thus, we have to provide a proper configuration object to initialize Atlas Blotter's row.

Use insertRow() or insertRows() from the blotter api property to add a new row.

grid.api.insertRow({
    ric: "JPY="
}); // Insert a new row at the bottom

grid.api.insertRow({
    ric: "EUR=",
    values: {"Field 1": 333, "Field 2": "abc"} // Initial static value
}, 0); // Insert a new row at the top (first row)

grid.api.insertRows([ // Insert multiple rows at once
    {},
    {ric: "aaa"},
    // ...
]);

All available row options can be found here

Adding data to Emerald Grid

Grid can be created with data by providing dataModel in the grid configuration object as described on the Initializing Data page. However, dataModel is entirely optional. You can add additional data to Grid at runtime by using APIs from Grid's DataTable or DataView. Adding data after initialization can be useful when you have some data from real-time sources or servers. Also, it allows you to define a row ID for referencing the row later.

Note: row index is not good for referencing as the row index will be shifted by various operations such as sorting, filtering, adding, and so on. It's recommended that you define the row ID (text) for each row, so that you can reference them later in a situation where you have some real-time data.

Use getDataTable() or getDataView() to retrieve Grid's internal data storage.

Use addRows() method to add multiple rows at once. The rows parameter is defined as an array of field/value paired objects as shown below: 

var dt = grid.api.getDataTable();
dt.addRows([
    {"field1": "a", "field2": 1, "field3": null},
    {"field1": "b", "field2": 5, "field3": {"someKey": 10}},
    {"field1": "c", "field2": 7, "field3": [1, 2, 3]},
    {"field1": "c", "field4": "some value"} // Rows don't need to be in uniform structure
]); // 4 rows are added

Use addRow() to add a single row to the bottom.

var dt = grid.api.getDataTable();
dt.addRow({"field1": "a", "field2": 1, "field3": null});

Use insertRow() if you want to add a row to a specific position. Note that you have to use DataView instead of DataTable this time, since the order of the row matters. DataView represents the final view of the grid after all manipulations are performed. The row index in DataView is the same row index shown in the grid UI, whereas the row index in DataTable is not. 

var dv = grid.api.getDataView();
dv.insertRow(2, {"field1": "a", "field2": 1}); // Add a new row as the third row to the top (row index #2)

For more details, see API Docs

Note: dataModel is completely optional for initilization
