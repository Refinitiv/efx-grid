Grid

Grid has been created to handle a large number of rows. It achieves high performance by utilizing a row virtualization technique, and all of the display logics and layouts are calculated programmatically. So, it lacks the flexibility of managing the layout when compared to the native table. With additional developments and enhancements, Grid has branched into several types. The degree of abstractions and purposes vary for each type as described below.

For more technical details, see Grid variations.

Atlas Blotter

Atlas Blotter is a custom element, using Lit library as its base implementation. It provides a quick way to show data in a grid layout with some basic functionalities. It also makes requesting and updating real-time data easier to do.

Installation

Install the element by the following command.

npm install @elf/atlas-blotter

The element is required theme to instantiate itself in the app. LSEG's design system is called Halo theme and you can install it from npm command.

npm install @elf/elf-theme-halo

Applications in LSEG Workspace should use ELF Halo themes to be fully compliant with branding guidelines. ELF requires a component's theme to initialize the component successfully in applications. See Element Framework for more details.

Setup guide

Use import syntax to import the Atlas Blotter, depended element and their themes into your app.

// import depended element and its theme
import '@elf/coral-icon';
import '@elf/elf-theme-halo/dark/imports/native-elements'; // import native styles for typography, css variables, etc.
import '@elf/elf-theme-halo/dark/coral-icon';

// import element and its theme
import '@elf/atlas-blotter';
import '@elf/atlas-blotter/themes/halo/dark/atlas-blotter';

Usage

Atlas Blotter can be easily created by just writing the tag and setting the configuration as shown below:

<body>
    <atlas-blotter></atlas-blotter>
    
    <script>
        var grid = document.getElementsByTagName("atlas-blotter")[0];
        grid.config = {
            /* See Real-time Grid section in the API Doc for available options */
        };
    </script>
</body>

Emerald Grid

Emerald Grid is a custom element, using Lit library as its base implementation. It is considered older version of Atlas Blotter as it lacks support for real-time data update.

Installation

Install the element by the following command.

npm install @elf/emerald-grid

The element is required theme to instantiate itself in the app. LSEG's design system is called Halo theme and you can install it from npm command.

npm install @elf/elf-theme-halo

Setup guide

Use import syntax to import the Emerald Grid, depended element and their themes into your app.

// import depended element and its theme
import '@elf/coral-icon';
import '@elf/elf-theme-halo/dark/imports/native-elements'; // import native styles for typography, css variables, etc.
import '@elf/elf-theme-halo/dark/coral-icon';

// import element and its theme
import '@elf/emerald-grid';
import '@elf/elf-theme-halo/dark/emerald-grid';

Usage

Create Emerald Grid by using emerald-grid tag and assign configuration to created element.

<body>
    <emerald-grid></emerald-grid>
    
    <script>
        var grid = document.getElementsByTagName("emerald-grid")[0];
        grid.config = {
            /* See Composite Grid section in the API Doc for available options */
        };
    </script>
</body>

Core Grid

Core Grid is a pure JavaScript-based component. Mostly it is responsible for managing UIs and data display logics. In general, you should not instantiate the Core Grid directly. Please use either Atlas Blotter or Emerald Grid instead.

Composite Grid and Real-Time Grid

The Composite Grid and Real-Time Grid are JavaScript wrappers that built on top of the Core Grid. The Core Grid is very flexible but only has a low-level API, which might be too complicated (due to its flexibility). The wrappers are created for developers looking to integrate the Grid quickly and easily into their application. With the Core Grid at its foundation, the wrappers inherits all of the great performance of the Core Grid.

Accessing Grid APIs

Use the api property from the grid element to access Composite Grid or Real-time Grid APIs. The api property will be available only after the config is set (i.e., configured event is fired).

Note: After or during configured event, the api property can then be used, as shown below:

var grid = document.getElementsByTagName("atlas-blotter")[0];
grid.addEventListener("configured", function (e) {
    var api = e.detail.api;
    var gridElem = e.detail.element;
    
    var core = api.getCoreGrid();
    var dv = api.getDataView();

    console.log("Grid has been configured.");
});

The API document for Emerald Grid can be found in the Composite Grid section.

The API document for Atlas Blotter can be found in the Real-Time Grid section.
